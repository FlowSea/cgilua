Overview

CGILua is a tool for creating dynamic HTML pages and manipulating input data from forms. It is simple but powerful, allowing complex tasks to be carried out with minimum effort. Typical applications that can be developed with CGILua include, for instance, WWW information systems based on external data (databases) and WWW gateways to existing systems.

CGILua uses the interpreted language Lua as a server-side scripting language for creating dynamic Web pages. Both pure Lua scripts and HTML templates are supported by CGILua. A Lua script is essentially a Lua program that creates the contents of a HTML page and sends it to the HTTP server. A HTML template is a conventional HTML file that embeds Lua code by using some special marks.

Lua scripts and HTML templates are equally easy to use, and choosing one of them basically depends on the characteristics of a specific page. While HTML templates are convenient for creating graphically sophisticated pages, probably with the help of a HTML editor, Lua scripts are more adequate for creating pages that are simple in terms of its HTML structure, but require a significative amount of internal processing. By also allowing these two methods to be intermixed, CGILua provides Web applications developers with great flexibility when both requirements are present. For a detailed description of both scripting methods and some examples of their use see CGILua scripts.

CGILua comes in five versions: Java Servlet, ISAPI module, Apache module, FastCGI and traditional CGI. CGILua's architecture is divided in two layers. The lowest one is represented by the launchers: The launchers implement a common set of functions also called the Server API. The highest layer is implemented over it and is totally portable between the launchers. This way any script made over the Server API could be run by any launcher.

CGILua is free software and uses the same license as Lua 5.0.

Incompatibility with previous versions

  • CGILua 5.0 uses Lua 5.0.
  • cgi table can has table values. See Receiving parameters for a more detailed explanation.
  • HTML templates marks are incompatible. See HTML Templates for a more detailed explanation.
  • The use of getenv to obtain CGI variables should be replaced by cgilua.servervariable.

Credits

CGILua 5.0

CGILua 5.0 was completely redesigned by Roberto Ierusalimschy, André Carregal and Tomás Guisasola as part of the Kepler Project. The implementation is compatible with Lua 5.0 and was coded by Tomás Guisasola with invaluable contributions by Ana Lúcia de Moura. CGILua 5.0 development was sponsored by Fábrica Digital and CNPq.

CGILua 4.0

Ana Lúcia de Moura adapted CGILua to Lua 4.0, reimplemented some code and added a few improvements but this version was not officially distributed.

CGILua 3.x

CGILua was born as the evolution of an early system developed by Renato Ferreira Borges and André Clínio at TeCGraf. At time there were no CGI tools, everything were done with shell scripts!

However, the main contribution was done by Anna Hester, who consolidated the whole tool and developed a consistent distribution with versions 3.1 and 3.2 (the number was an effort to accomplish Lua version number). This version was widely used by a great variety of systems.
